Add POST /auth/apple endpoint to auth.py router. Request schema AppleAuthRequest with fields: identity_token (str), full_name (FullName | None with given_name/family_name), email (str | None). Response schema AppleAuthResponse extends TokenResponse with is_new_user (bool).

The endpoint must:
1. Fetch Apple JWKS from https://appleid.apple.com/auth/keys (add helper function fetch_apple_public_keys with caching)
2. Decode identity_token JWT header to get kid
3. Verify signature using matching Apple public key (use PyJWT)
4. Validate claims: iss == "https://appleid.apple.com", aud == app bundle ID from settings, exp not expired
5. Extract sub (Apple user ID) from token payload
6. Find user by apple_id or create new user if not found
7. For new users: set apple_id, email (if provided), display_name from full_name
8. Issue access_token and refresh_token using existing helpers
9. Return AppleAuthResponse with is_new_user flag

Add to backend/src/backend/auth.py: fetch_apple_public_keys() async function that fetches and caches Apple's JWKS. Use httpx for the request. Cache for 1 hour.

Add APPLE_BUNDLE_ID to config.py Settings class (default: "com.slipstream.app").

Per D-08: tokens last 30 days (already configured via refresh_token_expire_days).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS App → Apple Auth | OS-level authentication, trusted |
| iOS App → Backend | Untrusted network, requires HTTPS + JWT validation |
| Backend → Apple JWKS | External dependency for token validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Apple identity token | mitigate | Server-side JWKS validation, verify iss/aud/exp claims |
| T-01-02 | Information Disclosure | Token storage | mitigate | Keychain with kSecAttrAccessibleAfterFirstUnlock, never UserDefaults |
| T-01-03 | Replay | JWT access token | mitigate | 15-min expiry, refresh token rotation on use |
| T-01-04 | Tampering | API requests | mitigate | HTTPS only, reject non-TLS connections |
| T-01-05 | Denial of Service | Token refresh | accept | Single refresh attempt per request, exponential backoff on repeated failures |
</threat_model>
