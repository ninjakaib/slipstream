## Interaction Patterns

### Onboarding Navigation

| Gesture/Action | Result |
|----------------|--------|
| Tap "Continue" | Animate to next step (slide left) |
| Tap back button | Animate to previous step (slide right) |
| Swipe | Disabled (programmatic navigation only per RESEARCH.md) |

**Source:** CONTEXT.md D-01, RESEARCH.md Pattern 4

### Username Validation

| State | Visual Feedback |
|-------|-----------------|
| Empty | No indicator |
| Typing (< 3 chars) | Small text: "At least 3 characters" in muted color |
| Checking | Spinner indicator |
| Available | Green checkmark + "Username available" |
| Taken | Red X + "This username is taken" |
| Invalid format | Red X + format error message |

**Source:** CONTEXT.md D-10

### Photo Selection

| Action | Result |
|--------|--------|
| Tap "Add Photo" | Show action sheet (Camera / Photo Library / Cancel) |
| Select from library | PhotosPicker opens, image loads into preview |
| Take photo | Camera opens, image loads into preview |
| Upload fails | Toast: "Photo upload failed. Tap to retry or skip for now." with action buttons |
| Tap "Skip" | Proceed to next step without photo |

**Source:** CONTEXT.md D-02, D-12

### Car Entry Cascading Pickers

| Action | Result |
|--------|--------|
| Select Year | Make picker populates, resets Model/Trim |
| Select Make | Model picker populates, resets Trim |
| Select Model | Trim picker populates (if trims exist) |
| Select "Other / Custom" | Free-text input appears |

**Source:** CONTEXT.md D-04, D-05

## Animation Specifications

| Animation | Duration | Curve | Description |
|-----------|----------|-------|-------------|
| Step transition | 300ms | easeInOut | Slide between onboarding steps |
| Progress dot | 200ms | spring | Scale/color change on step change |
| Validation indicator | 150ms | easeOut | Fade in checkmark/X |
| Toast appearance | 250ms | spring | Slide down from top |
| Toast dismissal | 200ms | easeIn | Fade out after 3 seconds |

**Source:** RESEARCH.md Pattern 4 (0.3s easeInOut), iOS standard conventions

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| VoiceOver labels | All buttons, indicators, and form fields have descriptive labels |
| Minimum touch targets | 44x44pt per iOS HIG |
| Dynamic Type | Support system text size scaling |
| Reduce Motion | Replace slide animations with crossfade |
| Color contrast | All text meets WCAG 2.1 AA (4.5:1 body, 3:1 large) |

## Visual Hierarchy

### Welcome Screen Focal Point

**Primary focal point:** Sign in with Apple button - positioned in lower third of screen, uses system-provided styling with accent outline, full-width touch target.

**Secondary focal point:** App name "SlipStream" and tagline centered in upper portion of screen using Display typography (28pt .semibold).

**Visual flow:** Logo/App name (top) -> Tagline (middle) -> Sign in with Apple button (bottom). Empty space used intentionally to create breathing room and direct attention to CTA.
