- @Binding var username: String
- var onNext: () -> Void
- var onBack: (() -> Void)?
- @State private var isAvailable: Bool? = nil
- @State private var isChecking: Bool = false
- @State private var errorMessage: String? = nil
- @FocusState private var isFocused: Bool

Validation logic using Combine:
- Create @StateObject private var validationDebouncer = UsernameValidator()
- UsernameValidator is @MainActor ObservableObject with:
  - @Published var username = ""
  - @Published var isAvailable: Bool? = nil
  - @Published var isChecking = false
  - @Published var errorMessage: String? = nil
  - private var cancellables = Set<AnyCancellable>()
  - private let apiClient: APIClient
  - init() sets up $username pipeline: .debounce(for: .milliseconds(500), scheduler: RunLoop.main), .removeDuplicates(), .filter { $0.count >= 3 }, then Task to checkAvailability
  - func checkAvailability(_ username: String) async — calls GET /users/search?q={username}, if exact match found (case-insensitive), isAvailable = false, else isAvailable = true

Per D-10: 500ms debounce, show checkmark/X indicator.

Body structure (per UI-SPEC):
- VStack with 24pt spacing
- Headline: "Choose a username" at 28pt semibold
- Subheadline: "This is how other drivers will find you" in muted color
- TextField with "@" prefix placeholder, bordered style
- ValidationIndicator below TextField showing:
  - Empty username: nothing
  - < 3 chars: Small text "At least 3 characters" in muted
  - Checking: ProgressView spinner
  - Available: Green checkmark SF Symbol + "Username available" in green
  - Taken: Red X SF Symbol + "This username is taken" in red
  - Invalid format: Red X + format error message
- Spacer
- PrimaryActionButton "Continue" disabled unless isAvailable == true
- No back button on first step (onBack is nil)

Username validation rules (match backend):
- 3-20 characters
- Alphanumeric + underscore only
- Pattern: ^[a-zA-Z0-9_]+$

Per D-03: Username is required, cannot proceed without valid available username.
